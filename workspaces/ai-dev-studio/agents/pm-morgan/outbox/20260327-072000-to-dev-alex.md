Implement per-agent executor selection for `AgentRunnerService`. This allows each agent to declare which execution backend it uses via an `executor` field in `agent.json`.

## Acceptance criteria

Read the full spec (user stories + acceptance criteria) at:
`M:/ai-dev-net/ai-dev-net/docs/specs/20260327-per-agent-executor-selection.md`

## Implementation checklist

Nova's architecture review has produced a concrete checklist. Implement the following:

1. **`AgentInfo`** — add an `Executor` string property (default `"claude"`). JSON deserialization via camelCase policy will bind the `executor` field automatically — no special reader needed.

2. **`AgentRunnerService` constructor** — change from `IAgentExecutor` to `IEnumerable<IAgentExecutor>`. Build an internal `Dictionary<string, IAgentExecutor>` keyed by `IAgentExecutor.Name` (case-insensitive).

3. **Executor resolution in `RunSessionAsync`** — read `executor` from `agentInfo` (default `"claude"` if absent). Look up in the dictionary. If not found, throw `InvalidOperationException` with the name requested and all available executor names listed. **No silent fallback to `"claude"`.**

4. **`Program.cs`** — remove the dead `BoardService` registration on line 15 (the factory registration on line 16–19 already overrides it; line 15 is dead code confirmed by a `// Remove this line` comment).

5. **No UI changes** — `Executor` is read-only in the UI for this release (PM decision). Do not add it to the meta-save form or the CreateAgent form.

## Scope boundaries

- No new executor implementations needed — `ClaudeAgentExecutor` is the only one. The change is purely in how it is resolved.
- The existing `IEnumerable<IAgentExecutor>` DI registration already works with a single `AddSingleton<IAgentExecutor, ClaudeAgentExecutor>()` call — no `Program.cs` registration changes needed beyond removing the dead line.
